# Lv.1 基礎: バックエンドの仕組みと概念

Lv.1のゴールは、<strong>「Webサーバーとデータベースの仕組み（Why/What）を正しく理解できること」</strong>です。
フレームワークを使う前に、HTTPプロトコルやデータベースの基礎的な概念を理解します。

## 1. WebとHTTPの仕組み
フロントエンドからのリクエストをどう受け取り、どう返すかを理解します。

### 学習項目
*   **HTTPプロトコル**:
    *   リクエストメソッド (GET, POST, PUT, DELETE) の使い分け。
    *   ステータスコード (2xx, 3xx, 4xx, 5xx) の意味。
    *   ヘッダー (Content-Type, Authorization) の役割。
*   **REST API設計**:
    *   リソース指向のURL設計 (`/users`, `/users/:id`)。
    *   ステートレスとは何か。

### チェックリスト
- [ ] GETとPOSTの違いを、副作用（Side Effect）の観点から説明できる。
- [ ] 401 (Unauthorized) と 403 (Forbidden) の違いを説明できる。
- [ ] クライアントにJSONを返す際、適切な `Content-Type` を設定できる。

## 2. データベースの基礎
データの永続化について学びます。

### 学習項目
*   **RDBMS (Relational Database)**:
    *   テーブル、カラム、レコードの関係。
    *   主キー (Primary Key) と外部キー (Foreign Key)。
    *   基本的なSQL (SELECT, INSERT, UPDATE, DELETE)。
*   **NoSQL (Optional)**:
    *   RDBMSとの違い（スキーマレス、スケーラビリティ）。

### チェックリスト
- [ ] 1対多、多対多のリレーションシップを図解できる。
- [ ] `WHERE`, `ORDER BY`, `LIMIT` を使った基本的なクエリが書ける。
- [ ] トランザクションが必要な場面（銀行の送金など）を説明できる。

## 3. 認証と認可
「誰が」「何をできるか」を管理する仕組みです。

### 学習項目
*   **認証 (Authentication)**:
    *   Cookie/Session方式とトークン方式 (JWT) の違い。
    *   パスワードのハッシュ化（平文保存の禁止）。
*   **認可 (Authorization)**:
    *   ロールベースアクセス制御 (RBAC)。

### チェックリスト
- [ ] なぜパスワードを暗号化（ハッシュ化）して保存する必要があるか説明できる。
- [ ] HTTPSを使わない通信でログイン情報を送る危険性を説明できる。
