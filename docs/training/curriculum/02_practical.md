# 02. 実践モジュール: アプリケーション開発

基礎知識を活用して、実際に機能するアプリケーションを作成します。ここでは「タスク管理アプリ（Todo App）」を題材にします。

## 1. プロジェクトのセットアップ
*   Vite を使用した React + TypeScript プロジェクトの作成。
*   ESLint / Prettier の設定。
*   ディレクトリ構成のベストプラクティス（`components`, `hooks`, `types`, `utils` など）。

## 2. アプリケーションの設計
コーディングを始める前に、設計を行います。

*   **コンポーネント設計**: UIをコンポーネントに分割し、階層構造を定義する。
*   **データ設計**: アプリケーションで扱うデータ（Todoアイテムなど）の型定義。
*   **状態管理設計**: どのコンポーネントがどの状態を持つべきか、あるいはグローバルな状態管理が必要か。

## 3. 実装ステップ

### ステップ 1: 静的なUIの作成
*   HTML/CSSで見た目を作成する。
*   コンポーネントへの分割。

### ステップ 2: 状態管理の実装
*   Todoリストの表示。
*   新しいTodoの追加。
*   Todoの完了/未完了の切り替え。
*   Todoの削除。

### ステップ 3: 永続化と副作用
*   `localStorage` を使ってデータを保存・読み込みする。
*   （オプション）Mock API (json-serverなど) との通信。

### ステップ 4: ルーティング (React Router)
*   詳細ページや設定ページへの遷移。

## 4. コード品質とリファクタリング
*   **カスタムフック**: ロジックの切り出しと再利用。
*   **パフォーマンス最適化**: `React.memo`, `useCallback`, `useMemo` の適切な使用。
*   **アクセシビリティ**: キーボード操作やスクリーンリーダーへの配慮。

## 課題
*   作成したTodoアプリに「フィルタリング機能（すべて/完了/未完了）」を追加する。
*   Context API または Redux/Zustand などのライブラリを使って、グローバルな状態管理を導入してみる。
