# 研修パーソナライズガイド

すべての研修生が同じバックグラウンドを持っているわけではありません。個々の経験やスキルに合わせて研修計画を調整することで、学習効果を最大化できます。

## 1. ターゲット別の調整方針

### ケースA: 完全な初心者（情報系卒だがWeb開発経験なし）
*   **特徴**: プログラミングの基礎概念（変数、ループ、条件分岐）は理解しているが、Webの仕組み（HTTP、DOM、ブラウザ）やJavaScript特有の非同期処理に慣れていない。
*   **重点領域**:
    *   **HTML/CSS**: 実際に画面を作る楽しさを知ってもらう。
    *   **JavaScript**: DOM操作やイベントハンドリング、非同期処理を丁寧に。
    *   **ツール**: Git, VS Code, ブラウザの開発者ツールの使い方。
*   **スキップ可能**: なし。すべてのモジュールを順を追って実施。
*   **メンタリング**: こまめなチェックインが必要。用語の定義から丁寧に説明する。

### ケースB: バックエンドエンジニア（サーバーサイド経験あり）
*   **特徴**: Java, Go, RubyなどでWebアプリケーションを作った経験がある。HTTPやDBの知識は豊富。
*   **重点領域**:
    *   **フロントエンド特有の概念**: コンポーネント指向、状態管理、レンダリングの仕組み。
    *   **CSS**: 苦手意識を持つ人が多いため、体系的に学ぶ（特にFlexbox/Grid）。
    *   **非同期処理**: サーバーサイドとは異なるJavaScriptのイベントループモデル。
*   **スキップ可能**: プログラミングの基礎（変数、関数など）。
*   **メンタリング**: サーバーサイドとの対比で説明すると理解が早い（例：「ReactのPropsは関数の引数のようなもの」）。

### ケースC: 経験豊富なJavaScript開発者（jQuery時代の経験など）
*   **特徴**: JavaScriptの経験は長いが、モダンなフレームワーク（React/Vue）やTypeScriptの経験がない。
*   **重点領域**:
    *   **宣言的UI**: 命令的（jQuery）な考え方からの脱却。DOMを直接操作しないこと。
    *   **TypeScript**: 型システムの導入メリットと実践的な型定義。
    *   **モダンなツールチェーン**: Webpack/Vite, Babel, ESLint などの役割。
*   **スキップ可能**: HTML/CSS/JSの基礎。
*   **メンタリング**: 「なぜモダンな手法が必要なのか」という背景（大規模開発への対応など）を説明し、納得感を醸成する。

## 2. 学習スタイルの違いへの対応

### 実践重視型（手を動かして学びたい）
*   **アプローチ**: 座学を最小限にし、すぐに「カウンターアプリ」や「Todoアプリ」の作成に入ってもらう。
*   **教材**: 公式ドキュメントのチュートリアルや、ハンズオン形式の記事。

### 理論重視型（体系的に理解したい）
*   **アプローチ**: 概念や仕組み（Reactのレンダリングサイクル、仮想DOMなど）をしっかり理解してからコードを書く。
*   **教材**: 体系的な書籍や、詳細な技術ドキュメント。

## 3. 調整のプロセス
1.  **事前ヒアリング**: 研修開始前にスキルセットや経験、学習スタイルの好みをヒアリングする。
2.  **計画のカスタマイズ**: 上記のガイドを参考に、カリキュラムの取捨選択や重点配分を行う。
3.  **柔軟な変更**: 研修の途中でも、進捗や理解度に合わせて計画を見直す。
